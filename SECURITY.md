# 安全策略

## 支持范围

当前仅对默认分支上的最新仓库状态提供安全修复支持。

## 报告方式

- 不要在公开 issue 中直接披露可被复现利用的安全细节。
- 请先提交一个最小公开描述，标记为安全问题，并避免给出可直接利用的 payload、号码或凭据。
- 维护者在接收后再决定是否转为私下处理或继续公开修复。

## 报告内容建议

- 影响范围。
- 触发条件。
- 复现步骤。
- 是否需要特殊权限。
- 是否涉及真实号码、通话记录或个人信息。

## 当前重点风险

- 权限滥用或误导性权限声明。
- 通话记录读取范围超出功能需要。
- 非预期的自动外呼行为。
- 文档与实现不一致导致的错误安全判断。
